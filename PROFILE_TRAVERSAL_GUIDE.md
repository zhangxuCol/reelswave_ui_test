# 个人中心页面遍历测试运行指南

## 快速开始

### 1. 运行测试

在项目根目录下执行以下命令：

```bash
python run_profile_traversal.py
```

### 2. 测试流程

测试脚本会自动执行以下步骤：

1. **打开浏览器**
   - 启动移动端模拟浏览器
   - 自动清理占用的端口
   - 设置为 iPhone Safari UA

2. **导航到个人中心页面**
   - 自动导航到个人中心页面
   - 等待页面加载完成

3. **保存页面HTML**
   - 将当前页面的HTML保存到 `html_files/profile.html`
   - 用于后续分析

4. **提取可交互元素**
   - 从HTML中提取所有可交互元素
   - 包括按钮、链接、输入框等

5. **遍历元素**
   - 依次点击每个可交互元素
   - 每次点击后返回到主页面
   - 继续测试下一个元素

6. **完成测试**
   - 所有元素测试完成后自动结束
   - 输出测试结果

## 测试结果

测试过程中会记录以下信息：

- 成功点击的元素
- 失败的元素及错误信息
- 跳过的已访问元素

所有日志会输出到控制台和日志文件中。

## 注意事项

1. 确保网络连接正常
2. 确保测试环境可访问
3. 测试过程中请勿手动操作浏览器
4. 某些元素可能导致页面跳转，框架会自动返回主页面

## 自定义配置

如需修改测试配置，可以编辑以下文件：

- `config/settings.py` - 修改基础URL
- `config/locators.py` - 修改元素定位器
- `page/page_traversal.py` - 修改测试逻辑
