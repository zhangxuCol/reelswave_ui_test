# 项目文档

本目录包含 ReelsWave Web 自动化测试项目的所有规范性文档和开发指南。

## 文档目录

1. [架构说明](architecture.md) - 项目整体架构和设计模式
2. [代码规范](coding_standards.md) - 代码编写规范和风格指南
3. [组件开发指南](component_guide.md) - 如何开发和维护页面组件
4. [测试编写指南](test_guide.md) - 如何编写和维护测试用例
5. [AI 指令模板](ai_prompts.md) - 使用 AI 开发时的标准指令模板
6. [命名规范](naming_conventions.md) - 文件、类、方法等命名规范
7. [最佳实践](best_practices.md) - 项目开发中的最佳实践

## 使用说明

### 对于开发人员

- 新加入团队的开发人员应先阅读 [架构说明](architecture.md) 和 [代码规范](coding_standards.md)
- 开发新功能前，请参考 [组件开发指南](component_guide.md) 或 [测试编写指南](test_guide.md)
- 遵循 [命名规范](naming_conventions.md) 和 [最佳实践](best_practices.md)

### 对于 AI 辅助开发

- 使用 AI 开发时，请参考 [AI 指令模板](ai_prompts.md)
- 确保 AI 生成的代码符合 [代码规范](coding_standards.md)
- 验证 AI 生成的代码是否遵循 [最佳实践](best_practices.md)

## 文档维护

- 保持文档的及时更新
- 重大架构变更时更新 [架构说明](architecture.md)
- 新增规范时更新 [代码规范](coding_standards.md)
- 定期审查和更新 [最佳实践](best_practices.md)
