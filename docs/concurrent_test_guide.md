
# 并发测试使用指南

## 概述

本指南说明如何使用 pytest-xdist 插件运行并发测试，以提高测试执行效率。

## 并发测试的优势

1. **提高执行速度**：多个测试用例同时运行，大幅减少总执行时间
2. **充分利用资源**：充分利用多核 CPU 的计算能力
3. **提高效率**：适合回归测试等需要频繁运行的场景

## 使用方法

### 方法一：使用并发测试脚本

#### 运行所有测试

```bash
# 使用默认4个进程运行所有测试
python run_concurrent_test.py

# 使用8个进程运行所有测试
python run_concurrent_test.py -n 8
```

#### 运行单个测试文件

```bash
# 使用默认4个进程运行个人中心测试
python run_concurrent_test.py page/profile_test.py

# 使用2个进程运行首页测试
python run_concurrent_test.py -n 2 page/home_test.py
```

### 方法二：直接使用 pytest

#### 运行所有测试

```bash
# 使用4个进程运行所有测试
pytest -n 4 -v -s

# 使用8个进程运行所有测试
pytest -n 8 -v -s
```

#### 运行单个测试文件

```bash
# 使用4个进程运行个人中心测试
pytest -n 4 page/profile_test.py -v -s

# 使用2个进程运行首页测试
pytest -n 2 page/home_test.py -v -s
```

#### 运行特定测试

```bash
# 使用4个进程运行所有冒烟测试
pytest -n 4 -m smoke -v -s

# 使用4个进程运行所有回归测试
pytest -n 4 -m regression -v -s

# 使用4个进程运行特定测试方法
pytest -n 4 page/profile_test.py::TestProfile::test_top_up_button -v -s
```

## 并发进程数选择

### 推荐的进程数

1. **CPU 核心数相同**
   - 4核 CPU：使用 4 个进程
   - 8核 CPU：使用 8 个进程

2. **CPU 核心数减 1**
   - 保留一个核心给系统使用
   - 4核 CPU：使用 3 个进程
   - 8核 CPU：使用 7 个进程

3. **根据测试类型调整**
   - UI 测试：建议使用较少进程（2-4个）
   - API 测试：可以使用更多进程（8-16个）

### 查看系统 CPU 核心数

```bash
# macOS/Linux
sysctl -n hw.ncpu

# Linux
nproc

# Windows
echo %NUMBER_OF_PROCESSORS%
```

## 注意事项

### 1. 测试独立性

确保测试用例之间相互独立：
- 不共享状态
- 不依赖执行顺序
- 每个测试都能独立运行

### 2. 资源竞争

避免资源竞争：
- 数据库连接
- 文件操作
- 浏览器实例

### 3. 报告生成

并发测试时：
- 每个进程生成自己的报告
- 最终会合并成一个报告
- 报告中的时间戳可能不准确

### 4. 调试困难

并发测试时：
- 错误信息可能不完整
- 难以复现问题
- 建议先串行运行确认问题

### 5. 网络限制

注意网络限制：
- 避免过多并发请求
- 可能触发服务器限流
- 考虑使用代理或限流

## 常见问题

### 问题1：测试失败率增加

**原因**：
- 并发导致资源竞争
- 网络请求过多
- 服务器限流

**解决方法**：
- 减少并发进程数
- 添加重试机制
- 使用测试数据隔离

### 问题2：浏览器实例冲突

**原因**：
- 多个进程同时使用同一浏览器
- 浏览器端口冲突

**解决方法**：
- 使用不同的浏览器端口
- 每个进程使用独立的浏览器实例
- 考虑使用无头模式

### 问题3：报告不准确

**原因**：
- 并发执行导致时间戳混乱
- 进程间通信问题

**解决方法**：
- 使用专门的报告插件
- 串行运行测试生成报告
- 使用日志文件辅助分析

## 最佳实践

1. **逐步增加并发数**
   - 从小并发数开始（2个进程）
   - 逐步增加找到最佳值
   - 监控系统资源使用情况

2. **定期串行验证**
   - 定期串行运行测试
   - 确保测试结果一致
   - 避免并发引入的问题

3. **使用测试分组**
   - 将测试按功能分组
   - 不同组使用不同并发策略
   - 优先级高的测试串行运行

4. **监控资源使用**
   - 监控 CPU 使用率
   - 监控内存使用情况
   - 监控网络流量
